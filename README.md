# পাঠশালা ই-ম্যানেজার - এপিআই ইন্টিগ্রেশন

## প্রকল্প পরিচিতি

পাঠশালা ই-ম্যানেজার একটি সম্পূর্ণ স্কুল ম্যানেজমেন্ট সিস্টেম যা এখন সম্পূর্ণ ডাইনামিক মোড সাপোর্ট করে। এই প্রকল্পে রেস্টফুল এপিআই এবং মঙ্গোডিবি ডাটাবেস ব্যবহার করা হয়েছে যা আধুনিক স্কুল ম্যানেজমেন্ট সিস্টেমের সকল প্রয়োজনীয়তা পূরণ করে। পূর্বে সিস্টেমটি শুধুমাত্র লোকাল স্টোরেজ ব্যবহার করত যার ফলে ডেটা শুধুমাত্র একটি ব্রাউজারে সংরক্ষিত থাকত এবং মাল্টি-ডিভাইস সাপোর্ট ছিল না। এখন নতুন আর্কিটেকচারে ক্লায়েন্ট এবং সার্ভার সম্পূর্ণ সেপারেট থাকায় যেকোনো প্ল্যাটফর্ম থেকে এপিআই অ্যাক্সেস করা সম্ভব এবং মোবাইল অ্যাপ বা অন্যান্য ক্লায়েন্ট তৈরি করা যাবে। এই ডকুমেন্টেশনটি ডেভেলপার এবং সিস্টেম অ্যাডমিনিস্ট্রেটরদের জন্য তৈরি করা হয়েছে যাতে তারা সিস্টেমটি সহজে বুঝতে এবং কাস্টমাইজ করতে পারেন।

## বৈশিষ্ট্য

### পুরাতন সিস্টেমের সীমাবদ্ধতা

পূর্বের স্ট্যাটিক সিস্টেমে অনেক সীমাবদ্ধতা ছিল যা একটি আধুনিক স্কুল ম্যানেজমেন্ট সিস্টেমের জন্য অগ্রহণযোগ্য। প্রথমত, সকল ডেটা ব্রাউজারের লোকাল স্টোরেজে সংরক্ষিত হতো যার অর্থ একটি ডিভাইসে যোগ করা ডেটা অন্য কোনো ডিভাইসে দেখা যেত না। এটি একটি বড় সমস্যা ছিল বিশেষত যখন একাধিক অ্যাডমিন বা শিক্ষক একই সিস্টেমে কাজ করতেন। দ্বিতীয়ত, ডেটা ব্যাকআপ এবং রিস্টোর করা কঠিন ছিল কারণ সকল ডেটা ব্যবহারকারীর ব্রাউজারে ছড়িয়ে ছিটিয়ে ছিল। তৃতীয়ত, রিয়েল-টাইম ডেটা সিঙ্ক সম্ভব ছিল না যা একটি আধুনিক সিস্টেমের জন্য অপরিহার্য।

### নতুন সিস্টেমের সুবিধা

নতুন ডাইনামিক সিস্টেমে এই সমস্যাগুলো সম্পূর্ণ সমাধান করা হয়েছে। এখন সকল ডেটা সেন্ট্রালাইজড মঙ্গোডিবি ডাটাবেসে সংরক্ষিত হয় যা যেকোনো জায়গা থেকে অ্যাক্সেস করা যায়। রিয়েল-টাইম ডেটা সিঙ্ক সমর্থন করে যাতে একই সাথে অনেক ব্যবহারকারী কাজ করতে পারেন এবং সকলের ডেটা আপডেট থাকে। অফলাইন সাপোর্টও রয়েছে যাতে ইন্টারনেট না থাকলেও কাজ চালিয়ে যাওয়া যায় এবং পরে স্বয়ংক্রিয়ভাবে সিঙ্ক হয়। এছাড়াও এন্টারপ্রাইজ-গ্রেড সিকিউরিটি ফিচার যেমন JWT অথেনটিকেশন এবং রোল-বেসড অ্যাক্সেস কন্ট্রোল যুক্ত হয়েছে।

## কুইক স্টার্ট গাইড

### পূর্বশর্ত

সিস্টেমটি চালাতে কিছু প্রাথমিক সেটআপ প্রয়োজন যা নিচে ধাপে ধাপে ব্যাখ্যা করা হয়েছে। প্রথমে আপনার মেশিনে নোডজেস এবং npm ইনস্টল করা থাকতে হবে। নোডজেসের ১৮ বা তার বেশি ভার্সন সাপোর্ট করে। এছাড়াও মঙ্গোডিবি ডাটাবেস ইনস্টল করা থাকতে হবে বা MongoDB Atlas একাউন্ট থাকতে হবে। আপনি যদি ক্লাউড ব্যবহার করতে চান তাহলে MongoDB Atlas ফ্রি টায়ার ব্যবহার করতে পারেন।

```bash
# নোডজেস ভার্সন চেক করুন
node --version  # v18+ প্রয়োজন

# npm ভার্সন চেক করুন
npm --version   # v9+ প্রয়োজন
```

### ধাপ ১: ডিপেন্ডেন্সি ইনস্টল করুন

প্রথমে প্রকল্পের মূল ফোল্ডারে যান এবং npm ইনস্টল কমান্ড চালান। এটি সকল প্রয়োজনীয় প্যাকেজ ইনস্টল করবে যা সার্ভার এবং ক্লায়েন্ট উভয়ের জন্য প্রয়োজন।

```bash
# প্রকল্প ফোল্ডারে যান
cd school-management-system

# সকল ডিপেন্ডেন্সি ইনস্টল করুন
npm install
```

### ধাপ ২: এনভায়রনমেন্ট কনফিগার করুন

`.env.example` ফাইলটি `.env` নামে কপি করে এবং আপনার কনফিগারেশন অনুযায়ী এডিট করুন। এই ফাইলে ডাটাবেস কানেকশন স্ট্রিং, JWT সিক্রেট কী এবং অন্যান্য সংবেদনশীল তথ্য থাকে যা গিট রিপোজিটরিতে কমিট করা উচিত নয়।

```bash
# .env ফাইল তৈরি করুন
cp .env.example .env
```

`.env` ফাইলে নিম্নলিখিত কনফিগারেশন যোগ করুন:

```env
# সার্ভার পোর্ট
PORT=3000

# মঙ্গোডিবি কানেকশন স্ট্রিং
MONGODB_URI=mongodb://localhost:27017/pathshala

# JWT সিক্রেট কী (নিরাপদ র‍্যান্ডম স্ট্রিং ব্যবহার করুন)
JWT_SECRET=your-super-secret-jwt-key-here

# এপিআই বেস ইউআরএল (ফ্রন্টএন্ড কনফিগ)
API_BASE_URL=http://localhost:3000/api

# সেশন সেটিংস
SESSION_SECRET=your-session-secret
```

### ধাপ ৩: সার্ভার চালু করুন

ডিভেলপমেন্ট মোডে সার্ভার চালাতে নিম্নলিখিত কমান্ড ব্যবহার করুন। এই কমান্ডটি Nodemon ব্যবহার করে যা কোড পরিবর্তনে স্বয়ংক্রিয়ভাবে সার্ভার রিস্টার্ট করে।

```bash
# ডিভেলপমেন্ট মোডে সার্ভার চালু করুন
npm run dev
```

সার্ভার সফলভাবে চালু হলে নিম্নলিখিত মেসেজ দেখাবে:

## এপিআই ব্যবহার গাইড

### বেস এপিআই ক্লায়েন্ট

নতুন `js/api.js` ফাইলে সম্পূর্ণ এপিআই ক্লায়েন্ট রয়েছে যা সকল CRUD অপারেশন সাপোর্ট করে। এই ক্লায়েন্ট অ্যাসিঙ্ক/অয়েট সিনট্যাক্স ব্যবহার করে এবং সহজে ব্যবহারযোগ্য মেথড প্রদান করে। প্রতিটি এনটিটি যেমন স্টুডেন্ট, টিচার, ক্লাস ইত্যাদির জন্য আলাদা মডিউল রয়েছে।

```javascript
// সকল স্টুডেন্ট লোড করুন
const students = await API.students.getAll();

// আইডি দিয়ে স্টুডেন্ট খুঁজুন
const student = await API.students.getById('student-id');

// নতুন স্টুডেন্ট তৈরি করুন
const newStudent = await API.students.create({
    firstName: 'আব্দুল',
    lastName: 'রহমান',
    banglaName: 'আব্দুল রহমান',
    classId: 1,
    status: 'active'
});

// স্টুডেন্ট আপডেট করুন
await API.students.update('student-id', { status: 'active' });

// স্টুডেন্ট মুছুন
await API.students.delete('student-id');

// সার্চ করুন
const results = await API.students.search('আব্দুল');
```

আপনি যদি পূর্বে তৈরি পেজগুলো আপডেট করতে চান তাহলে নিম্নলিখিত পরিবর্তনগুলো করুন। প্রথমে HTML ফাইলে নতুন স্ক্রিপ্ট ফাইলগুলো যোগ করুন এবং তারপর জাভাস্ক্রিপ্ট কোড আপডেট করুন।


### লাইভ উদাহরণ

`admin/admin-students.html` ফাইলটি একটি সম্পূর্ণ উদাহরণ যা দেখায় কীভাবে নতুন এপিআই সিস্টেম ব্যবহার করতে হয়। এই ফাইলটি রেফারেন্স হিসেবে ব্যবহার করে অন্যান্য পেজগুলোও আপডেট করতে পারেন। প্রতিটি ফাংশন স্বতন্ত্র এবং পুনরায় ব্যবহারযোগ্য করে লেখা হয়েছে যাতে কোড ডুপ্লিকেশন কমে যায় এবং মেইনটেইনেন্স সহজ হয়।

## ডিবাগিং এবং ট্রাবলশুটিং

### সাধারণ সমস্যা সমাধান

কিছু সাধারণ সমস্যা দেখা দিতে পারে যা সহজে সমাধান করা যায়। সার্ভার কানেক্ট না হলে প্রথমে চেক করুন সার্ভার চলছে কিনা এবং পোর্ট সঠিক কিনা। CORS এরর দেখা দিলে সার্ভারের CORS কনফিগারেশন চেক করুন। 401 এরর মানে অথেনটিকেশন সমস্যা, লগইন স্ট্যাটাস চেক করুন।

```javascript
// কানেকশন টেস্ট
async function testAPIConnection() {
    try {
        const response = await fetch(API_CONFIG.baseURL.replace('/api', '/health'));
        console.log('সার্ভার স্ট্যাটাস:', response.ok ? 'অনলাইন' : 'সমস্যা');
        return response.ok;
    } catch (error) {
        console.error('কানেকশন ব্যর্থ:', error.message);
        return false;
    }
}

// কনসোলে চালান
testAPIConnection();
```

### লগিং এবং মনিটরিং

সার্ভার সাইড লগিং এবং ফ্রন্টএন্ড কনসোল লগ ব্যবহার করে ডিবাগ করুন। সার্ভার টার্মিনালে সকল রিকোয়েস্ট এবং এরর লগ হয়। ব্রাউজার কনসোলে API কলের রেসপন্স দেখা যায়।

## প্রোডাকশন ডিপ্লয়মেন্ট

### বিল্ড এবং অপ্টিমাইজেশন

প্রোডাকশনে ডিপ্লয় করার আগে কিছু অতিরিক্ত ধাপ প্রয়োজন। সবচেয়ে ভালো পারফরম্যান্সের জন্য মিনিফাইড এবং অপ্টিমাইজড বিল্ড ব্যবহার করুন। এছাড়াও এনভায়রনমেন্ট ভেরিয়েবল সঠিকভাবে সেট করুন এবং HTTPS এনেবল করুন।

```bash
# প্রোডাকশন বিল্ড
npm run build

# PM2 দিয়ে সার্ভার চালান
pm2 start server/index.js --name pathshala-api

# সার্ভার স্ট্যাটাস চেক
pm2 status

# লগ দেখুন
pm2 logs pathshala-api
```

### নগিনক্স কনফিগারেশন

প্রোডাকশনে নগিনক্স দিয়ে রিভার্স প্রক্সি সেটআপ করুন যা বেটার পারফরম্যান্স এবং সিকিউরিটি প্রদান করে।

```nginx
server {
    listen 80;
    server_name api.yourdomain.com;
    
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

## ভবিষ্যৎ উন্নয়ন

### পরিকল্পিত ফিচার

সিস্টেমটি ক্রমাগত উন্নত করা হচ্ছে এবং কিছু নতুন ফিচার পরিকল্পিত রয়েছে। মোবাইল অ্যাপ আসছে যাতে অ্যাডমিন এবং শিক্ষকরা মোবাইল থেকে সিস্টেম ব্যবহার করতে পারেন। SMS এবং ইমেইল নোটিফিকেশন সিস্টেম যুক্ত হবে যাতে অভিভাবকদের স্বয়ংক্রিয়ভাবে জানানো যায়। অ্যানালিটিক্স ড্যাশবোর্ড যুক্ত হবে যাতে বিভিন্ন রিপোর্ট এবং চার্ট দেখা যায়। মাল্টি-ল্যাংগুয়েজ সাপোর্ট যুক্ত হবে ভবিষ্যতে।
